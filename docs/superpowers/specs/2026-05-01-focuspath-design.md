# FocusPath — Design Spec

**Date:** 2026-05-01  
**Status:** Approved

## Overview

FocusPath is a standalone Next.js web app that trains ADHD users to build longer attention spans through daily reading of spiritual texts. Users read a curated passage, have their focus time measured, then must pass a 3-question comprehension gate to complete the session. Progress is tracked and graphed over time so users can see their attention span growing.

---

## Goals

- Force genuine engagement with texts (not skimming)
- Make attention span improvement visible and motivating
- Leverage spiritual content as inherently calming and focus-inducing
- Deploy to Vercel; no account required to start (auth optional for persistence)

## Screens

### 1. Dashboard (`/`)
- Greeting with user's name
- Today's passage card (title, quote preview, estimated read time) — tapping starts the session
- Today's passage is selected deterministically by date (`hash(date) % passages.length`) so all users on the same day see the same passage; filtered by the user's enabled traditions
- Three stat pills: Avg Focus Time, Total Sessions, Quiz Score %
- Weekly attention span bar chart (Mon–Sun, today highlighted)
- Streak badge in nav bar

### 2. Reading (`/session/[id]`)
- Source label (e.g. "Marcus Aurelius · Meditations")
- Progress bar (scroll-based, 0–100%)
- Full passage text in Georgia serif
- Live focus timer (counts up from 0:00)
- Target time shown alongside timer
- "I've finished reading" button — only enabled after 60% scroll and minimum 60 seconds

### 3. Comprehension Gate (`/session/[id]/quiz`)
- 3 multiple-choice questions generated by Claude API based on the passage
- Questions shown one at a time
- Correct answer revealed after each choice with a brief explanation
- Must score 2/3 or better to mark session complete
- On failure: shown which answers were wrong, offered to re-read (not penalized on streak)

### 4. Progress (`/progress`)
- Big headline: current average focus time
- Delta vs. week 1 (e.g. "↑ +4m 32s from week 1")
- Weekly average focus time bar chart (last 6 weeks)
- Breakdown by text source: name, sessions completed, mini progress bar

### 5. Nav
- Bottom nav bar (mobile-first): Home · Today · Progress · Settings
- Settings: choose text sources (toggle each tradition on/off), set daily target time

## Content

Spiritual texts stored as static JSON in `/data/passages/`. No external API for content.

**Sources included:**
- Bhagavad Gita
- Upanishads
- Tao Te Ching
- Stoics (Marcus Aurelius, Epictetus, Seneca)
- Bible (Psalms, Proverbs, Sermon on the Mount)
- Quran (selected verses)
- Buddhist sutras (Dhammapada)

Each passage object:
```json
{
  "id": "stoics-001",
  "source": "Marcus Aurelius",
  "text": "Meditations",
  "quote": "You have power over your mind...",
  "body": "Full passage text here...",
  "estimatedMinutes": 8,
  "tradition": "stoics"
}
```

Comprehension questions generated at session start via Claude API (`claude-haiku-4-5`) and cached in DB per passage.

## Data Model (Prisma + Neon PostgreSQL)

```prisma
model User {
  id          String       @id @default(cuid())
  email       String?      @unique
  name        String?
  sessions    Session[]
  settings    UserSettings?
  createdAt   DateTime     @default(now())
}

model UserSettings {
  id              String   @id @default(cuid())
  userId          String   @unique
  user            User     @relation(fields: [userId], references: [id])
  traditions      String[] @default(["stoics","gita","tao","upanishads","bible","quran","buddhist"])
  dailyTargetMins Int      @default(8)
}

model Session {
  id           String   @id @default(cuid())
  userId       String
  user         User     @relation(fields: [userId], references: [id])
  passageId    String
  focusSeconds Int
  quizScore    Int      // 0-3
  completed    Boolean  @default(false)
  date         DateTime @default(now())
}

model QuizCache {
  passageId   String   @id
  questions   Json     // array of {question, choices[4], correctIndex, explanation}
  createdAt   DateTime @default(now())
}
```

Sessions stored locally in `localStorage` for unauthenticated users; synced to DB on sign-in.

## Key Behaviors

- **Minimum read time:** 60 seconds + 60% scroll before "Done" button enables — prevents instant-skip
- **Comprehension gate:** 2/3 correct required to complete session; re-read is always available
- **Streak:** Increments when a session is completed (`completed: true`) on a given calendar day
- **Progress chart:** Weekly average of `focusSeconds` across all completed sessions that week
- **Quiz caching:** Claude-generated questions cached in DB per `passageId` — not regenerated each time
